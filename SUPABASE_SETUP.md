# SUPABASE_SETUP.md
# Lumi — Supabase Setup Guide
# ============================
# One-time setup. Takes about 10 minutes.
# Free tier is plenty for beta (500MB, 50k monthly active users).

---

## Step 1 — Create your Supabase project

1. Go to https://supabase.com and sign in (GitHub login is easiest)
2. Click "New project"
3. Name it: `lumi-beta`
4. Choose a strong database password (save it somewhere safe)
5. Region: **Europe (Frankfurt)** — closest to South Africa with good latency
6. Click "Create new project" — takes ~2 minutes to provision

---

## Step 2 — Create the database tables

Go to **SQL Editor** in the left sidebar and run each block below.

### 2a — Users table

```sql
create table users (
  id           text primary key,           -- UUID from localStorage
  display_name text,                       -- child's first name, optional
  created_at   timestamptz not null,
  last_seen    timestamptz,
  is_claimed   boolean default false,      -- true once linked to real email
  email        text,                       -- null until claimed
  constraint users_id_length check (char_length(id) between 32 and 36)
);

-- Index for quick lookups
create index users_last_seen_idx on users(last_seen desc);
```

### 2b — Sessions table

```sql
create table sessions (
  id         text primary key,             -- generated by db.js
  user_id    text references users(id),
  week       integer not null,             -- 1–8
  subject    text default 'afrikaans',     -- afrikaans | english | maths
  started_at timestamptz not null,
  ended_at   timestamptz,                  -- null if still active
  constraint sessions_week_check check (week between 1 and 52)
);

create index sessions_user_idx on sessions(user_id, started_at desc);
```

### 2c — Sentence results table

```sql
create table sentence_results (
  id             bigserial primary key,
  user_id        text references users(id),
  session_id     text references sessions(id),
  week           integer not null,
  sentence_index integer not null,          -- 0–6 within a week
  mode           text not null,             -- listen | tap | play
  correct_words  integer default 0,
  total_words    integer default 0,
  completed      boolean default false,
  duration_secs  integer default 0,
  recorded_at    timestamptz not null,
  constraint results_mode_check check (mode in ('listen','tap','play'))
);

create index results_user_idx   on sentence_results(user_id, recorded_at desc);
create index results_session_idx on sentence_results(session_id);
```

### 2d — Feedback table

```sql
create table feedback (
  id         bigserial primary key,
  user_id    text references users(id),
  message    text not null,
  created_at timestamptz not null default now()
);

create index feedback_created_idx on feedback(created_at desc);
```

## Step 3 — Set up Row Level Security (RLS)

This is important — it controls who can read and write what.
Run this in the SQL Editor:

```sql
-- Enable RLS on all tables
alter table users            enable row level security;
alter table sessions         enable row level security;
alter table sentence_results enable row level security;
alter table feedback         enable row level security;

-- Users: anyone can insert/update their own row (matched by id)
create policy "users_insert" on users
  for insert with check (true);

create policy "users_update" on users
  for update using (true);

-- Sessions: anon users can insert their own sessions
create policy "sessions_insert" on sessions
  for insert with check (true);

create policy "sessions_update" on sessions
  for update using (true);

-- Sentence results: anon users can insert
create policy "results_insert" on sentence_results
  for insert with check (true);

-- Feedback: anyone can insert
create policy "feedback_insert" on feedback
  for insert with check (true);

-- Read: users can only read their own data
-- (You'll tighten this when you add real auth)
create policy "users_read_own" on users
  for select using (true);   -- relax for now; tighten with auth later

create policy "sessions_read_own" on sessions
  for select using (true);

create policy "results_read_own" on sentence_results
  for select using (true);

-- Sentence results: allow delete (for progress reset)
create policy "results_delete_own" on sentence_results
  for delete using (true);

-- Sessions: allow delete (for progress reset)
create policy "sessions_delete_own" on sessions
  for delete using (true);

```

If you already ran the RLS setup earlier, run only these two policies in the SQL Editor:
```sql
create policy "results_delete_own" on sentence_results for delete using (true);
create policy "sessions_delete_own" on sessions for delete using (true);
```

**If "Reset progress" doesn't clear stars/dots after refresh:** the delete policies above are required. Run them in Supabase → SQL Editor.

**For feedback:** run the feedback table and policy from section 2d and Step 3.

## Step 4 — Get your API keys

1. In Supabase, go to **Settings → API**
2. Copy:
   - **Project URL** — looks like `https://abcdefgh.supabase.co`
   - **anon public key** — long JWT string starting with `eyJ...`

3. Open `db.js` and replace the two placeholder values at the top:

```js
const SUPABASE_URL     = 'https://YOUR_PROJECT_ID.supabase.co';  // ← paste here
const SUPABASE_ANON_KEY = 'YOUR_ANON_PUBLIC_KEY';                 // ← paste here
```

## Step 5 — Wire up user.js and db.js in your app

Add to the top of your `app.js` (or `index.html` if not using modules):

```js
import { initUser, setDisplayName } from './user.js';
import { initDB, startSession, endSession, recordSentenceResult } from './db.js';
```

Then in your app initialisation (wherever you currently set up state):

```js
// 1. Init user first (sync, instant)
const user = initUser();

// 2. Init DB in background (async, non-blocking)
initDB().catch(console.warn);

// 3. Start a session when a week is selected
let currentSession = null;

async function onWeekSelected(weekNumber) {
  // End previous session if one is active
  if (currentSession) await endSession(currentSession.id);
  currentSession = await startSession({ week: weekNumber, subject: 'afrikaans' });
}

// 4. Record a result when a sentence is completed
// Call this inside your existing sentenceComplete() function:
async function onSentenceComplete({ sentenceIndex, mode, correctWords, totalWords, durationSecs }) {
  if (currentSession) {
    await recordSentenceResult({
      sessionId:     currentSession.id,
      week:          state.unitIndex + 1,
      sentenceIndex,
      mode,
      correctWords,
      totalWords,
      completed:     true,
      durationSecs,
    });
  }
}
```

## Step 6 — Optional: Name prompt UI

Add this anywhere in your app to let parents name the child's profile.
(Suggested: show it on first visit after they complete their first sentence.)

```js
import { setDisplayName } from './user.js';
import { updateDisplayName } from './db.js';

function showNamePrompt() {
  const name = prompt("Wat is jou naam? 🌟");  // Or build a proper modal
  if (name) {
    setDisplayName(name);           // saves locally
    updateDisplayName(name);        // syncs to Supabase
  }
}

// Trigger after first sentence completion:
if (user.isNew && state.completed.size === 1) {
  showNamePrompt();
}
```

4. You should see POST requests to `*.supabase.co`
5. In Supabase → Table Editor → `sentence_results`, you should see a row appear

## Viewing your data

In Supabase → Table Editor you can browse rows directly.

For quick stats, run these in the SQL Editor:

```sql
-- Daily active users
select date(last_seen), count(*) as users
from users
group by 1 order by 1 desc limit 14;

-- Sentences completed per week
select week, count(*) as completions
from sentence_results
where completed = true
group by 1 order by 1;

-- Average score per week (listen mode only)
select week,
  round(avg(correct_words::float / nullif(total_words,0) * 100)) as avg_pct
from sentence_results
where mode = 'listen' and total_words > 0
group by 1 order by 1;

-- Most active users
select u.display_name, u.id, count(r.id) as sentences_done
from users u
join sentence_results r on r.user_id = u.id
group by 1,2 order by 3 desc limit 20;
```

## When you're ready for real accounts

The schema is already ready. You'll just:
1. Enable Supabase Auth (built-in)
2. Add a "Claim this profile" flow that links the existing UUID to an email
3. Update the RLS policies to use `auth.uid()` instead of `true`

Nothing in the data model changes — all historical data carries forward.
